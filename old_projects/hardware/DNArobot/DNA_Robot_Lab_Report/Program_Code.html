Explanation:


This program uses a simple combination of variables, a nested-for loop, a lookup command, and four if/else statements. It iterates through a user-defined DNA sequence, and analyzes it to determine the appropriate movement.


The symbol commands simply renames the variables b0 and b1 to more relevant references. Two variables are declared: counter and basepair. Counter will be used later with the main loop, and basepair will store each DNA sequence character, in turn. All of the code beyond this point is placed within a nested for loop.


The loop should be defined to continue for the number of characters there are in the sequence. In this case, two characters were defined, so the loop would repeat two times. The range set in the loop declaration specifies the number of times that the commands within should loop. In this example, for counter = 0 to 1, 0 is considered a part of the range. The program will run each command in the loop, in sequence. When it reaches the next counter command, the loop increases the counter variable by one, and if the new value for counter is within the specified loop range, it returns the program to the beginning of the loop, repeating all of the commands declared within.


The first command within the loop is a lookup command. It reads the nth character of the quoted alphanumeric string. For example, if the counter variable currently has a value of 0, it will read the first character in the string; if counter has a value of 3, it will read the third character in the string. Lookup always reads left to right. The string must be enclosed, first, in double quotations, then in round brackets. After the string is declared, the lookup command will store the character it has read in the variable that follows (in this case, basepair). The next time the program loops, lookup will read the next character in the string and overwrite the value of the basepair variable with the new character’s value. Numeric characters are stored into variables as is, but alphabetical characters are automatically converted to their ASCII equivalent when they are stored. To make a reference to these values, the ASCII decimal equivalent must be used (refer to http://www.asciitable.com/ for decimal equivalent for each character).


The command set that follows is composed of a series of if/else statements that analyze the current value of the basepair variable. The initial condition is declared: whether basepair is equal to the ASCII equivalent for G. If it is, the program will run the commands specified within the if statement command structure. If it is not, it will move onto the next condition, which checks the basepair variable with the ASCII equivalent for C. This process continues until the program has analyzed for each valid input type (G,C,A,T). The endif command declares the end of the statement.


When an if statement is satisfied, the program will run a series of commands that allow certain inputs to the H-Bridge to be set to high or low. This, in turn, causes the motor(s) (and wheels) to spin a particular direction, and ultimately, allows the robot to move or turn in a particular direction. The high command will set the output pin that is referenced to HIGH, and vice-versa for the low command. The wait command tells the program to wait before doing anything further, for the specified time (unit: seconds, integers only).
