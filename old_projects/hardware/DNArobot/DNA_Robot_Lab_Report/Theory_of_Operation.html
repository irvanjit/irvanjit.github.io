H-Bridge Circuit


The H-Bridge circuit has a simple function. It allows the output received by a motor to be switched, allowing for reversible direction that the motor spins in, while providing enough current to power the motor .The circuit has two inputs (commonly referred to as D0 and D1), two TIP31C transistors, two TIP42C transistors, and two outputs. Refer to the H-Bridge schematic for specifics on wiring the circuit. If the D0 input is given a High voltage, and D1 input is given a Low voltage, then the circuit will transfer the current in one direction, and electricity will flow from D0 into the circuit, to one output and into another, then back through the circuit into D1 (Low) (one output gets a High voltage, and the other gets a Low voltage). If the voltages given to D0 and D1 are switched (that is, D0 is given a Low voltage and D1 is given a High voltage), then the current at the output end of the circuit will also switch direction of flow. Essentially, the circuit allows the output current to be reversed by switching the D0 and D1 voltage levels. In this application of the H-Bridge circuit, a simple motor is connected to the outputs of the H-Bridge. The motors used for this robot are designed such that when the motor is given High on one end, and Low on the other, the motor will spin in one direction, and when the voltage levels on the motor are switched, the direction that the motor spins will also switch. When the motor is attached to the H-Bridge circuit, the direction that the motor is spinning can be controlled simply by switching the input voltages at D0 and D1.


Robot Structure


The DNA Robot’s structure consists of a chassis with two motors attached to it. Each motor is connected directly to a wheel. This entire frame is mounted securely onto a breadboard with two wired H-Bridge circuits, with wires connecting the motors to the circuits. Each circuit at controls the direction of the motor it is connected to. This setup allows for the robot to move like most land vehicles; move forwards and backwards, and turn left and right. To move forwards, both wheels would spin in one specific direction, and the opposite direction to travel backwards. To turn left, the left motor would need to stop turning (high would be given to both D0 and D1, disabling that circuit), and to turn right, the right wheel would have to stop turning. To stop the robot, both of the circuits would have to be disabled. Therefore, the robot’s movements are controllable by sending different combinations of voltages to the D0 and D1 inputs.


PICAXE Microprocessor


The PICAXE 28X1 Microprocessor provides the robot’s intelligence. The PICAXE controls the direction that the robot travels in. The D0 and D1 inputs for both of the H-Bridge circuits are connected to four of the PICAXE’s output pins. The PICAXE can be programmed such that the D0/D1 inputs receive different voltages from the PICAXE’s outputs, allowing for variable and controllable movement.


Programming:


The last four output pins on the PICAXE 28X1 ( outputs 4,5,6,7)  are used to provide the voltage levels to the D0/D1 inputs. The voltage level at any output pin can be controlled using the following command:


high/low x


x is an output pin reference from 0 to 7 (specific to 28X1). The following command can be used to set either a high or low voltage (on or off) to pin x




To take DNA code as input and output it as movement of the robot, the following components are required: a nested for loop and a lookup command, to iterate through and read/store the DNA sequence values, and a series of if/else statements to define the movements when different letters are read.


Here are examples of each type of command/command sets:


Nested For Loop:


for b1 = 0 to 4

execute code placed here

next b1


b1 is a declaration of a variable which can store data in the PICAXE’s memory. This structure will allow the program to execute whatever is within the for loop; when the program reaches next b1, it is told to return to the beginning of the loop, and use the next value for b1. Declaring that b1 = 1 to 4 specifies that the first time the program loops, b1 will equal 0, for the second loop, b1 will equal 1, and so on, until b1 = 4, it will stop after this, looping for a total of five times.



If/Else Statement:


b1 = 4

if b1 = 5    then

execute code placed here

else if  b1 < 5then

execute code placed here

endif


the variable b1 is equal to 4. An if or if/else statement has three main components. The If/else if/endif commands that are italicised: these tell the program to check for the condition that follows after, and to stop checking at the end; the condition: this defines what the program is checking for, in this case, it first checks if b1 is equal to five. Since it is not, it moves to the next if else  statement, checking if b1 is less than 5. This is true; the last part of an if statement defines the action that the program should do if a statement turns out to be true or false, depending on the conditions/statements specified.



Lookup Command:

**Important



for b1 = 0 to 4

lookup b1, (“ABCDE”) ,b2

next b1


The first line causes the lookup command to loop five times. For each time that the program loops, the last line, next b1, increases the value of the variable b1. For the first loop, b1 = 0, for the second loop, b1 = 1, and so on, until b1 = 4. The lookup command tells the program to read the character at the Nth position within the quoted string ABCDE and store the value of the character into the variable b2. If b1 = 0, then lookup will read the first character in the string, “A”, and store the ASCII decimal equivalent of the character “capital A” into b2. If b1 = 1, then it will store “B” into b2. Since the program loops for the number of letters in the sequence, all letters are read and temporarily kept in the variable b2 where they can be used.
