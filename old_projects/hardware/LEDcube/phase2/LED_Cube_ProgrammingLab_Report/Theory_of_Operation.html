Theory of Operation

 

General Theory

The cube consists of 4 LED ‘levels’. Each level has 4 rows and 4 columns of LEDs, all attached by their anodes and cathodes with thin rigid metal wires. An LED is lit when its cathode is given a low voltage, and its anode is given a high voltage. This allows for electricity to flow from the positive end, through the LED and out through the cathode. Likewise, giving the cathode rows a low voltage and the anode rows a high voltage will light all of the LEDs. The anodes and cathodes are all connected with CAT5e wire which is color coded.


On the other aspect of this setup, the breadboard, a PICAXE 28X1 is mounted onto a breadboard, along with four 74LS245 ICs. The PIC serves as the controller for the entire circuit: In terms of connections, the complete system schematic and breadboard pictorial diagrams are ample descriptions as to how the circuit should be wired. The CAT5e cable connects the four anodes (+) and four cathodes (-) of each level to the four 74LS245 chips (each IC should have four + and - connections as outputs). The first eight output pins for the 28X1 are connected to each of the ICs in their designated locations. In order to accomplish this, the cables must be jumped (i.e. the first output connects to the first input for IC1, IC2, IC3 and IC4, and so on, for all 8 PIC outputs). The two pins after that are to power the PIC, and the last four pins serve as enable output for the ICs, they are each connected to the first output pin on each of the ICs.


Recall that giving a cathode low voltage and an anode high voltage will turn on the LED. Similarly, when the first output pin on each 74LS245 is given a low voltage, the entire IC is enabled or “turned on” , and when a  high voltage is given, the IC is disabled. Therefore, the PICAXE, which connects to each anode and cathode row of each level in the cube, all via the four ICs, is able to control the entire system. If the last four output pins of the PICAXE send a low voltage, the ICs will be enabled, and if they send a high, the ICs will be disabled. Sending a high/low from each of the anode-connected pins and cathode connected pins will turn on or off specific LEDs, according to the voltages sent.


The PICAXE also has input pins which are connected to a specific pins on the serial connector that will communicate with the programming computer. The input pins also have connected various resistors, which are necessary to regulate the electricity flowing in certain locations to appropriate voltage levels, and the also have an LDR variable resistor connected, which changes the amount of voltage that one of the input pins receives, depending on the lighting conditions that the LDR is in. Refer to the pictorial diagram under the “Diagrams” section of the lab report for details as to how the breadboard is wired.

PRogramming THeory/BASIC

When code is programmed to the PICAXE, it gets stored in the PICAXE’s built-in memory. This allows for the cube, once powered, to operate without constantly being connected to the computer. The act of programming the cube itself requires the serial connector to be plugged into a computer with a compatible DB9 port, and must also be wired to the PICAXE (refer to Pictorial Diagram, the serial connector is at the bottom of the diagram and has a total of 9 pins on it). The cube must also be powered. This is most easily accomplished using the USB cable made in the previous lab. Refer to the Pictorial for how to wire this). Once all of this wiring is complete, the cube can be programmed.


The actual code used to program the PICAXE is really straightforward. The PICAXE microprocessor is programmed in the BASIC programming language, and using the Picaxe Programming Editor software. The program uses typical basic structures and commands. It is recommended that you refer to the PICAXE Starting Documentation and the Datasheet for the 28X1 chip, particularly the “BASIC Commands” documentation before beginning to program. The documentation can be found on the PICAXE website (http://www.rev-ed.co.uk/picaxe/), under the “Datasheets” tab (found at the top of the page). The commands are relatively straightforward, and for simple programs, there is no prior programming knowledge required.


To start any program, the ’main:’ command is used. At the end of the program, the goto main command can be added; this will cause the program to run through the code that is put between main and goto main commands, and then return to the main command.


Example:


main:            starts the program


all of the code for the program is placed here


goto main        loops back to the main command, creating an infinite cycle.


To get an LED to turn on, it is a simple matter of enabling the IC chip that connects to the level that the particular LED is located in, and the giving the anode row of the LED a HIGH value, and the cathode column a LOW value. By default, pins 0 through 8 on the PICAXE (the first eight pins) default to LOW, however, it is a good practice to still declare this, as the PICAXE may have HIGH value stored as an output for some of the pins from a previous program, so leaving it alone will leave the value as HIGH, thus keep the column disabled.


To enable an IC chip, the ‘voltagevalueportc pinreference’ command is used, where the voltage value can either be a high or a low, high to disable and low to enable. To turn the row or column on, the command ‘voltagevalue pinreference’ is used. To confirm specific pin reference numbers, consult the datasheet for the 28X1.


Example:

this code will turn on one LED in one level


main:


    low portc 4            enables the IC chip connecting to portc pin 4

    high portc 5

    high portc 6

    high portc 7        disables the IC chips connecting to c pins 5-7


*recall that enabling or disabling an IC chip effectively turns the entire LED level on or off

   


        low 0            enables the column that connects to pin 0

        high 1

        high 2

        high 3        disables the columns connecting to pins 1-3


       

        high 4        enables the row that connects to pin 4

        low 5

        low 6

        low 7            disables the rows that connect to pins 5-7



goto main                returns to main



Similar code structure can be used to program the cube for a variety of applications. The pause command is another common command that is followed by an integer, and causes the program to wait for the specified interval in milliseconds. There are many other commands that can be used, but for the purposes of this lab, the commands covered above are sufficient for most of the programs.


Another major concept that is vital for programming the PICAXE connected to LEDs is knowledge and understanding of demultiplexing. Demultiplexing is when an LED is turned on, then after waiting for a very small period of time (i.e. 1ms), the LED is turned off. This process is repeated for multiple LEDs, and then the entire command set is looped. The end result is that all of the LEDs included in this demultiplexed series will all appear to be on at the same time. They are really all just taking turns to flash on and off, but this happens so fast that what we see is a clean and stable image.


Example 1: Without Demultiplexing


main:


    low portc 4,5,6,7            enables all levels


    low 0,1,2,3                enables all columns

    high 4,5,6,7            enables all rows



Example 2: With Demultiplexing


main:


    low portc 4,5,6,7            enables all of the IC chips



        low 0           

        low 1

        low 2

        low 3                enables all of the LED columns


       

        low 4

        low 5

        low 6

        low 7                disables all of the LED rows


        high 4            enables row 4

        pause 1            waits for 1 millisecond

        low 4                disables row 4


        high 5            enables row 5

        pause 1            waits for 1 millisecond

        low 5                disables row 5


        high 6            enables row 6

        pause 1            waits for 1 millisecond

        low 6                disables row 6


        high 7            enables row 7

        pause 1            waits for 1 millisecond

        low 7                disables row 7


*even though the rows are simply taking turns to flash on and off, they will appear to be on at the same time because of the extremely small pause value that separates the flashes


goto main                returns to main, allowing for demultiplexing



In the above Examples, both programs have the same outcomes. The key difference is that the second example uses demultiplexing, and does not simultaneously keep all of the rows and columns enabled, but cycles between enabling and disabling them. The purpose of doing this is when a specific pattern of LEDs is required to be turned on (i.e. a diagonal line), it must be used in order to obtain the correct results. Simply enabling the correct rows and columns will not work. Demultiplexing allows reduces the power consumption of the cube, and greatly reduces the load on the processors, which are capable, but not nearly meant for processor-intensive applications.

This value was then assigned to a variable, and then used to create a program that would vary in display depending on lighting conditions. Observations:


There were some small errors in some of the programs when they were first developed and programmed to the PICAXE.
